## Полином

**Файл:** [polynom.c](polynom.c)

Составьте программу `polynom.c`, вычисляющую значение полинома

```
P_n(x) = a_n*x^n + a_(n-1)*x^(n-1) + ... + a_1*x + a_0
```

и его производной в заданной точке x₀. Коэффициенты полинома и значение x₀ — целые числа в диапазоне от −2⁶³ до 2⁶³ − 1.

Программа считывает из стандартного потока ввода степень полинома n, значение x₀ и коэффициенты a_n, ..., a_0. Выводит значения P_n(x₀) и P_n'(x₀).

Для вычисления использовать **схему Горнера**:

```
P_n(x) = (...((a_n*x + a_(n-1))*x + a_(n-2))*x + ... + a_1)*x + a_0
```

Например: `P_4(x) = 3x⁴ + 2x³ − 5x² + x − 4 = (((3x + 2)x − 5)x + 1)x − 4`

## Произведение чисел по модулю

**Файл:** [mulmod.c](mulmod.c)

Составьте программу `mulmod.c`, вычисляющую выражение `(a·b) mod m` — остаток от деления произведения чисел a и b на m. Известно, что a, b и m — натуральные числа, меньшие 2⁶³, причём m ≠ 0.

Основная сложность: произведение a·b может превышать 2⁶⁴ и не помещаться ни в один целочисленный тип языка Си.

Решение сводится к вычислению значения полинома по схеме Горнера, представив число b в двоичной системе счисления:

```
(a·b) mod m = [(...(a·b₆₃·2 + a·b₆₂)·2 + ... + a·b₁)·2 + a·b₀] mod m
```

При вычислении использовать тождества: `(x+y) mod m ≡ ((x mod m)+(y mod m)) mod m` и `(x·y) mod m ≡ ((x mod m)·(y mod m)) mod m`.

## Фибоначчиева система счисления

**Файл:** [fibsys.c](fibsys.c)

Числа Фибоначчи — последовательность {fᵢ}, где f₀ = f₁ = 1, fᵢ = fᵢ₋₂ + fᵢ₋₁ для i > 1.

По теореме Цекендорфа любое положительное целое число может быть единственным образом представлено в виде суммы различных чисел Фибоначчи, не включающей двух подряд идущих.

**Алгоритм разложения** числа x в сумму Цекендорфа (пока x > 0):
- находим максимальное число Фибоначчи f ≤ x
- добавляем f в сумму Цекендорфа
- вычитаем f из x

**Фибоначчиева система счисления** — позиционная система с двумя цифрами (0 и 1), базисом которой являются числа Фибоначчи, начиная с f₁: 1, 2, 3, 5, 8, 13, 21, 34, ...

Составьте программу `fibsys.c`, выполняющую перевод целого числа 0 ≤ x < 2⁶³ в фибоначчиеву систему счисления.

**Файл:** [intersect.c](intersect.c)

Программа считывает размер и элементы множества A, затем размер и элементы множества B, и выводит элементы A ∩ B, отсортированные по возрастанию.

Использовать массивы для хранения множеств **запрещается**: каждое множество должно быть представлено 32-разрядным целым числом, где i-й бит равен 1, если число i принадлежит множеству.

## Функция поиска в лисповском списке

**Файл:** *(нет решения)*

В гипотетическом интерпретаторе языка Lisp элемент списка представлен структурой `Elem`:

```c
struct Elem {
    /* Тег, описывающий тип значения в «голове» списка */
    enum {
        INTEGER,
        FLOAT,
        LIST
    } tag;

    /* Само значение в «голове» списка */
    union {
        int i;
        float f;
        struct Elem *list;
    } value;

    /* Указатель на «хвост» списка */
    struct Elem *tail;
};
```

В элементе списка может храниться целое число, число с плавающей точкой или указатель на список. Тип определяется полем `tag`. Если элемент последний в списке, `tail = NULL`.

```c
struct Elem *searchlist(struct Elem *list, int k)
{
    ...
}
```

Функция возвращает указатель на найденный элемент или `NULL`, если элемент не найден.

> **Указание.** Добавьте в файл `#include "elem.h"` или целиком скопируйте определение структуры `Elem` из условия задачи.
