## Стековая машина

**Файл:** [stackmachine.c](stackmachine.c)

**Стековая машина** — устройство для выполнения арифметических операций, использующее стек целых чисел для хранения промежуточных результатов.

Составьте программу `stackmachine.c`, моделирующую работу стековой машины.

Поддерживаемые операции:

- `CONST x` — кладёт в стек число x (−1 000 000 000 < x < 1 000 000 000)
- `ADD` — снимает a и b, кладёт a + b
- `SUB` — снимает a, затем b, кладёт a − b
- `MUL` — снимает a и b, кладёт a · b
- `DIV` — снимает a, затем b, кладёт результат целочисленного деления a на b
- `MAX` — снимает a и b, кладёт max(a, b)
- `MIN` — снимает a и b, кладёт min(a, b)
- `NEG` — меняет знак числа на вершине стека
- `DUP` — кладёт копию числа на вершине стека
- `SWAP` — меняет местами два числа на вершине стека
- `END` — завершает работу, выводит число на вершине стека

**Формат результата:** число, оставшееся на вершине стека после выполнения всех операций.

## Слияние последовательностей

**Файл:** [merge.c](merge.c)

**Формат входных данных:**
- Первая строка: количество массивов k
- Вторая строка: размеры массивов n₁, n₂, ..., nk
- Каждая из следующих k строк: элементы соответствующего массива

**Пример:**
```
4
3 5 1 2
10 12 20
15 16 17 19 25
20
11 12
```
→ `10 11 12 12 15 16 17 19 20 20 25`

> **Указание.** Вместо нескольких параллельных массивов создайте один массив структур.

## Моделирование работы вычислительного кластера

**Файл:** *(нет решения)*

Имеется вычислительный кластер из N однопроцессорных узлов. Нужно выполнить M задач. Каждая задача описывается парой (t₁, t₂), где t₁ — время готовности входных данных, t₂ — прогнозируемое время выполнения.

Порядок выполнения задач нельзя менять: если данные для задачи A готовы раньше, чем для задачи B, то A не может быть запущена позже B.

Составьте программу `cluster.c`, вычисляющую минимальное время от начала работы кластера до завершения всех задач. В программе использовать **очередь с приоритетами** для хранения времён окончания задач.

**Формат входных данных:**
- Первая строка: количество узлов N
- Вторая строка: количество задач M
- Каждая из следующих M строк: пара (t₁, t₂), отсортированная по возрастанию t₁, t₁ уникальны

**Формат результата:** минимальное время завершения всех задач в микросекундах.

## Лексический анализ

**Файл:** *(нет решения)*

Определения:
- **Константа** — непустая последовательность десятичных цифр
- **Специальный знак** — один из символов: `+`, `-`, `*`, `/`, `(`, `)`
- **Идентификатор** — непустая последовательность латинских букв и цифр, начинающаяся с буквы
- **Лексема** — константа, специальный знак или идентификатор

Для каждой лексемы вычисляется пара (tag, value):
- `const` — значение константы как десятичное число
- `spec` — порядковый номер в списке `+, -, *, /, (, )` (начиная с нуля)
- `ident` — порядковый номер первого вхождения идентификатора в предложении (начиная с нуля)

**Пример:** `alpha + x1 (beta alpha) x1 y` → alpha=0, x1=1, beta=2, y=3

Составьте программу `lexavl.c`, выполняющую лексический анализ предложения.

**Формат входных данных:** первая строка — размер предложения n, вторая строка — само предложение.

> **Указание.** Использовать АВЛ-дерево для хранения отображения идентификаторов в их значения. Строку можно считывать посимвольно через `getc()` или `fgetc()`.

- `END` — последняя команда

> **Указание.** Хеш-функция: h(i) = i mod m.

## Количество подпоследовательностей, на которых побитовое ИСКЛЮЧАЮЩЕЕ ИЛИ даёт ноль

**Файл:** *(нет решения)*

Пусть дана последовательность из n целых чисел, где 0 < n ≤ 100 000, каждое в диапазоне от −2³¹ до 2³¹ − 1. Составьте программу `zeroxor.c`, вычисляющую количество подпоследовательностей, у которых побитовое ИСКЛЮЧАЮЩЕЕ ИЛИ всех элементов равно 0.

**Пример:** для a = {0, 14, 14, 2, 2, 0} таких подпоследовательностей 10:
- a[0:0], a[0:2], a[0:4], a[0:5], a[1:2], a[1:4], a[1:5], a[3:4], a[3:5], a[5:5]

> **Указание.** Пусть XOR подпоследовательностей [0:i] и [0:j] равен x, и не существует k, что i < k < j и XOR [0:k] равен x. Тогда количество подпоследовательностей с правой границей j и XOR равным 0 на единицу больше, чем с правой границей i.
